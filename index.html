Design your Programming Language

What a programming language needs to be better of others?

Requirements to Fly:

  1. High-level programming language permits to develop faster on large projects.
  2. Compiled language programming is faster than interpreted.
  3. A solution which implement all programming paradigms (Object Oriented, Procedural and Functional) guarantees to chose better ways in different cases.
  4. A new project without reinvent the wheel.
  5. A clean and easy code syntax, guessable from average programmer.
  6. Take the best features from all know programming languages, to own all you need.
  7. Integrate natively most know design patterns to be one step ahead.

Project Plan

This planning explain how the work is organized in order have a clear vision of the project

Phases

Project is defined in 5 phase, some can run on parallel others need the completion of the previous one.

  1. Define the Syntax of the language
  2. Build the Compiler
  3. Equip with standard and auxiliary code Libraries
  4. Add Utility & Tools
  5. Write Documentation

Management

Every phase will be divided in tasks and in sub-tasks if necessary, but all works will be agreed from community. Each step to new development must consider advantages and disadvantages by discussing on Community channels.

The Syntax

This is the first phase and the most important because will determine the programming language usability. It's preferable not to upset and not to invent strange things, but use what a programmer already know. New features will be added in the next releases.
Consider the following features taken from known programming language:

  • Duck Typing from Python
  • Overloading Operators from Python and C++
  • Package import management like Python or Go
  • Aggregator Type already integrated like Python and Rust
  • Template like C++, or Generic Type like Java
  • Visibility scope and Functional Programming like Java
  • Object Oriented and Procedural like Python and C++
  • GC management like Go
